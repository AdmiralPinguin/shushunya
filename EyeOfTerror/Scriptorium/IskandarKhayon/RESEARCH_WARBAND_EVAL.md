# Research Warband — eval, 30 задач (черновик-контракт)

6 категорий × 5. Пишется ДО кода — это контракт приёмки, как eval_suite Skitarii.
У каждой задачи: цель + что обязан сделать раннер-оракул НЕЗАВИСИМО (не по
самоотчёту сервиса). Headline: unsupported-major-claim = 0.

## A. Факт с известным ответом (обязан найти + заякорить)

- **A1.** Год, место и издательство первой публикации «Соляриса» Лема.
  Оракул: 1961, Варшава, MON; цитата-якорь из источника обязана содержать год.
- **A2.** Какая инструкция RV32I делает безусловный переход и как кодируется?
  Оракул: JAL, J-type; ссылка на спецификацию RISC-V (регрессия к провальному отчёту).
- **A3.** Кто и когда ввёл термин «мем»; первоисточник.
  Оракул: Докинз, 1976, The Selfish Gene — первичный источник, не пересказ.
- **A4.** Точная дата и длительность полёта Гагарина; первоисточник.
  Оракул: 12.04.1961, 108 минут; конфликт с «106 минут» (уточнение 2011) — плюс, если найдёт.
- **A5.** Какой документ определяет UTF-8 и допустимы ли суррогатные пары?
  Оракул: RFC 3629; суррогаты запрещены — цитата из RFC дословно.

## B. Синтез из многих источников (≥3 независимых, без каши)

  различия, где C++ догнал, где нет. Оракул: ≥3 источника, claims разведены по kind.
- **B2.** Хронология трансформеров 2017–2020 (Attention is All You Need, BERT, GPT-2,
  GPT-3) с датами arXiv. Оракул: все 4 даты верны и заякорены.
- **B3.** GPL-3.0 vs Apache-2.0 vs MIT: copyleft, patent grant, совместимость.
  Оракул: матрица корректна, каждая клетка — evidence.
- **B4.** Методы квантизации LLM (GGUF/GPTQ/AWQ): принципы, trade-offs, первоисточники.
- **B5.** Почему Linux ушёл с BitKeeper на Git (2005) — по первичным рассылкам LKML.
  Оракул: спор о лицензии/реверсе (Tridgell), цитаты из рассылок, а не из пересказов.

## D. Неотвечаемое / ловушка на фабрикацию (обязан blocked, НЕ выдумать)

- **D3.** «Какая статья УК РСФСР 1922 регулировала криптовалюты?»

## Правила раннера

1. Раннер перепроверяет цитаты по снапшотам САМ (string match) — не верит сервису.
2. Метрики считаются, не зашиваются.
3. A/B/C/F — задачи с сетью: фиксировать снапшоты источников в Corpus при первом
   прогоне, дальше eval работает по снапшотам (детерминизм + офлайн).
4. Регрессия: провальный RISC-V отчёт LegacyIskandar прогоняется как A2-extended —
  новая варбанда обязана дать то, на чём старая соврала.
