# ShushunyaOS VM

Изолированная заготовка будущей ОС Шушуни. Это отдельная VM рядом с
`skitarii-vm`; она не использует её диски, порты, NVRAM или PID.

Сейчас здесь только инфраструктура:

- Q35 + KVM + UEFI/OVMF;
- 16 vCPU и 32 ГиБ RAM при будущем запуске;
- пустой системный qcow2 на 80 ГиБ;
- пустой архивный qcow2 на 200 ГиБ;
- изолированная user-mode сеть и будущий SSH на `127.0.0.1:2223`;
- без GPU passthrough, общих папок и переноса данных проекта.

`24.04` — номер выпуска Ubuntu/Pop!_OS, а не версия ядра. Базовая ветка
ядра намеренно пока не выбрана.

## Команды

```bash
./scripts/prepare-vm.sh
./scripts/verify-vm.sh
./scripts/status-vm.sh
```

`prepare-vm.sh` создаёт только пустые thin-provisioned образы и личный UEFI
NVRAM. Он ничего не форматирует и не запускает.

`start-vm.sh` сейчас намеренно откажется запускаться. Позже, после установки
системы, запуск будет разблокирован явным маркером `runtime/vm/os-installed`.
Диск данных по умолчанию не подключается; для конкретного запуска потребуется
явный аргумент `start-vm.sh --with-data`.

Исходящие соединения гостя по умолчанию запрещены через `restrict=on`.
Будущий канал к LLM на физическом хосте будет добавлен отдельно и только для
нужного API. Перед первым реальным запуском QEMU также нужно вынести под
отдельного малопривилегированного пользователя хоста.
