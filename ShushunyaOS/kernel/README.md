# Kernel workspace

Исходники ядра и конкретная базовая ветка пока намеренно не выбраны.

Когда начнётся работа над ядром:

1. фиксируем поддерживаемую upstream/LTS-ветку;
2. сохраняем исходный конфиг и точный commit;
3. каждое изменение ведём отдельным патчем в `patches/`;
4. собираем в `build/`, не меняя ядро физического хоста;
5. проверяем только внутри `shushunya-os-vm` со снимком системного диска.

Каталог `source/` и результаты `build/` исключены из Git.
