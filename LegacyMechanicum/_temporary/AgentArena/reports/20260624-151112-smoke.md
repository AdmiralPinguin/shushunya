## Findings

The initial failed Shushunya results were mostly harness problems, not agent
failures:

- Shushunya is intentionally constrained to sandbox paths like `/work`; the
  first harness asked it to write into `/media/...`, which conflicts with its
  own system prompt.
- The harness could not write `/work` directly as the Codex user. It now uses
  the sandbox host mapping under
  `/media/shushunya/ARCHIVE/shushunya-agent-sandbox/work` through
  `sg shushunya-agent`.
- A false negative came from `test -f -- path`; shell `test` does not accept
  `--` there. The sandbox file existed and content checks already passed.

Real signal from this smoke run:

- Shushunya can solve both tasks correctly in its own sandbox.
- Shushunya is currently much slower on simple tasks than Aider and mini-SWE.
- Aider and mini-SWE are worth studying for lean command/edit loops.
- OpenHands needs Docker/Podman before it can be fairly benchmarked locally.
